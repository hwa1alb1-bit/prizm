# Service Readiness Evidence

This directory stores sanitized Branch 4: service readiness archives generated by:

```bash
pnpm verify:service-readiness
```

The command writes a timestamped JSON file with:

- authenticated `/api/ops/health` status and connector evidence,
- Vercel production deployment proof,
- Stripe webhook, delivery, Customer Portal, and credit-grant proof,
- DNSSEC DS and Cloudflare template reconciliation,
- GitHub repository governance state,
- dashboard-only exceptions with owner and next proof step.

Use `SERVICE_READINESS_ALLOW_INCOMPLETE=1 pnpm verify:service-readiness` only when archiving a known partial run. Partial archives must keep every dashboard-only item assigned to an owner with a concrete next proof step.

Do not store auth cookies, API keys, webhook secrets, or dashboard screenshots in this directory. Attach sensitive screenshots to the private evidence store and reference the evidence ID in the JSON or monthly SOC 2 pack.
